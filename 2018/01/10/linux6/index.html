Linux就该这么学（六）

发表于 2018-01-10 | 分类于 Technology

Linux作为一个十分优秀的操作系统，优秀的程序员应该熟练的掌握Linux，let’s do it.孰能生巧。

Linux就该这么学(六)

远程控制服务

ssh服务

SSH(Secure Shell)是一种能够以安全的方式提供远程登录的协议，其格式为 ssh[参数] 主机IP地址。

  • 若果想要禁止root管理员的身份远程登录到服务器。则可以使用 vim /etc/ssh/sshd_config, 之后修改48行PermitRootLogin。

  • 安全密钥认证：1、使用 ssh-keygen 生成密钥对；2、使用 ssh-copy-id 将生成的公钥文件传送至远程主机；3、使用 vim /etc/ssh/sshd_config, 修改78行 PasswordAuthentication。

远程传输命令

scp（secure copy）是一个基于SSH协议在网络之间进行安全传输的命令， 其格式为 scp[参数] 本地文件 远程账户@远程IP地址：远程目录

参数 作用
-v 显示详细的连接进度
-P 指定远程主机的sshd端口号
-r 用于传送文件夹
-6 使用IPv6协议

此外，还可以使用scp命令把远程主机上的文件下载到本地主机，其格式为 scp[参数] 远程用户@远程IP地址：远程文件 本地目录

不间断会话服务

screen是一款能够实现多窗口远程控制的开源服务程序，简单来说就是为了解决网络异常中断或为了同时控制多个远程终端窗口而设计的程序。还可以使用screen服务程序同时在多个远程会话中自由切换。

  • 会话恢复：即便网络中断，也可以让会话随时恢复，确保用户不会失去对远程会话的控制。

  • 多窗口：每个会话独立运行的，拥有各自独立的输入输出终端窗口。

  • 会话共享：当多个用户同时登录到远程服务器时，便可以使用会话共享功能让用户之间的输入输出信息共享。

管理远程会话

screen命令可以做很多事情。

参数 作用
-S 创建会话窗口
-d 指定会话进行离线处理
-r 恢复指定会话
-x 一次性恢复所有的会话
-ls 显示当前所有的会话
-wipe 把目前无法使用的会话删除

会话共享功能

screen命令还具有会话共享、分屏切割、会话锁定等功能。

  • 终端A，ssh一台地址
  • screen一个新屏幕
  • 终端B，ssh一台地址
  • screen -x
    通过以上步骤，即可实现共享，两台主机可以看到同一个屏幕。

使用vstfpd服务传输文件

文件传输协议

FTP是一种在互联网中进行文件传输的协议，基于客户端/服务器模式，默认使用20、21号端口。其中，FTP协议有如下两种工作模式。

  • 主动模式：FTP服务器主动向客户端发起连接请求。
  • 被动模式：FTP服务器等待客户端发起连接请求

vstpd是一款运行在Linux操作系统的FTP服务程序。安装过后，iptables防火墙管理工具默认禁止了FTP传输协议的端口号，因此，使用如下命令。

iptables -F
service iptables save

vsftp一些常见的参数以及作用

参数 作用
listen=[YES\NO] 是否以独立运行的方式监听服务
listen_address=IP地址 设置要监听的IP地址
listen_port=21 设置FTP服务的监听端口
download_enable=[YES\NO] 是否允许下载文件
userlist_enable=[YES\NO] userlist_deny=[YES\NO] 设置用户列表为“允许”还是“禁止”操作
max_clients=0 最大客户端连接数，0为不限制
max_per_ip=0 同一IP地址的最大连接数，0为不限制
anonymous_enable=[YES\NO] 是否允许匿名用户访问
anon_upload_enable=[YES\NO] 是否允许匿名用户上传文件
anon_umask=022 匿名用户上传的umask值
anon_root=/var/ftp 匿名用户的ftp根目录
anon_mkdir_write_enable=[YES\NO] 是否允许匿名用户创建目录
anon_other_write_enable=[YES\NO] 是否开放匿名用户的其他写入权限
anon_max_rate=0 匿名用户的最大传输速率，0为不限制
local_enable=[YES\NO] 是否允许本地用户登录FTP
local_umask=022 本地用户上传的umask值
local_root=/var/ftp 本地用户的FTP根目录
chroot_local_user=[YES\NO] 是否将用户权限禁锢在FTP目录，以确保安全
local_max_rate=0 本地用户最大传输速率，0为不限制

vsftpd服务程序

vsftpd作为更加安全的文件传输的服务程序，允许用户以三种认证模式登录到FTP服务器上。

  • 匿名开方模式：是一种最不安全的认证模式，任何人都可以无需密码验证而直接登录到FTP服务器。

  • 本地用户模式：是通过Linux系统本地的账户密码信息进行认证的模式。

  • 虚拟用户模式：是这三种模式中最安全的一种认证模式，它需要为FTP服务单独建立用户数据库支持，虚拟出用来进行口令验证的账户信息。

使用Samba或NFS实现文件共享

Samba文件共享服务

用于创建Samba的参数以及作用

参数 作用
[database] 共享名称为database
comment = Do not arbitrarily modify the database file 警告不要随意修改数据库
path = /home/database 共享目录
public = no 关闭 “所有人可见”
writable = yes 允许写操作
  • 第一步：创建用于访问共享资源的账户信息。

pdbedit命令用于管理SMB服务程序的账户信息数据库，格式为 pdbedit[选项] 账户。

参数 作用
-a 建立Samba账户
-x 删除Samba账户
-L 列出账户列表
-LV 列出账户详细信息列表 
pdbedit -a -u 用户名
  • 第二步：创建用于共享资源的文件目录。考虑到SELinux安全上下文所带来的限制。最后使用restorecon使之生效。
mkdir /home/databse
chown -Rf ck:ck /home/database
semanage fcontext -a -t samba_share_t /home/databse
restorecon -Rv /home/databse

  • 第三步：设置SELinux服务与策略。

  • 第四步：在Samba服务程序的主配置文件加入信息。

  • 第五步：重启smd服务。

NFS（网络文件系统）可以使得远程Linux系统上的文件共享资源挂载到本地主机的目录上，从而使得本地主机基于TCP/IP协议访问。
